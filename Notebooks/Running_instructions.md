# Running Instructions

## Recommended Environment

The notebook was developed and validated in Kaggle Notebooks.

- **Notebook:** `vehicle-type-classification-cnn-transfer-learning.ipynb`
- **Public notebook:** [Kaggle version 11](https://www.kaggle.com/code/danielariasroyo/vehicle-type-classification-cnn-transfer-learning)
- **Dataset:** [Stanford Cars Dataset by Classes Folder](https://www.kaggle.com/datasets/jutrera/stanford-car-dataset-by-classes-folder)
- **Accelerator:** GPU recommended (NVIDIA T4 was used)
- **Internet:** Enable Internet so pretrained ImageNet weights and required packages are available

## Run on Kaggle

1. Open the public Kaggle notebook or create a new Kaggle Notebook.
2. If creating a new notebook, import:

   ```text
   Notebooks/vehicle-type-classification-cnn-transfer-learning.ipynb
   ```

3. Add the Kaggle dataset:

   ```text
   jutrera/stanford-car-dataset-by-classes-folder
   ```

4. In the notebook settings, enable a GPU accelerator and Internet access.
5. Confirm that the dataset is available at:

   ```text
   /kaggle/input/datasets/jutrera/stanford-car-dataset-by-classes-folder
   ```

6. Select **Run All** and execute the notebook from top to bottom.

The notebook is stateful. Running isolated later sections after restarting the Kaggle session can produce `NameError` messages because models, predictions, or helper objects have not yet been created.

## Notebook Workflow

The sequential workflow is:

1. Import libraries and set the random seed
2. Configure dataset and output paths
3. Explore the dataset and class distribution
4. Create training, validation, and test datasets
5. Apply image resizing, preprocessing, and augmentation
6. Train and evaluate the custom CNN
7. Train and evaluate MobileNetV2
8. Train and evaluate ResNet50
9. Compare model accuracy, loss, parameters, and training time
10. Generate classification reports and confusion matrices
11. Analyze correct and incorrect ResNet50 predictions
12. Generate Grad-CAM explanations
13. Generate SHAP explanations
14. Save summary tables, figures, and trained models

## Common Issues

### Dataset path not found

Confirm that the dataset has been added to the Kaggle notebook and that `BASE_DIR` matches the mounted dataset path.

### Variables are not defined

Use **Run All** after a session restart. In particular, Grad-CAM sections `17.4` through `17.7` must run sequentially.

### SHAP plotting error for multiple outputs

The notebook already splits the final SHAP output dimension into a list before calling `shap.image_plot`. Do not replace section `18.7` with a direct `shap.image_plot(shap_values)` call when using the validated Kaggle SHAP version.

### GPU memory or cache limitations

Restart the Kaggle session and run from the beginning. If memory remains limited, reduce `BATCH_SIZE` from `32` to `16` or remove `.cache()` from the dataset optimization cell.

## Run Locally

Local execution is possible with Python, TensorFlow, NumPy, pandas, Matplotlib, Pillow, scikit-learn, and SHAP. Download and extract the dataset, then update `BASE_DIR` in section `2` to the local dataset directory. A CUDA-compatible GPU is strongly recommended.

- Validation split: `20%` of the official training folder
- Test set: official Stanford Cars test folder
- Transfer learning weights: ImageNet
