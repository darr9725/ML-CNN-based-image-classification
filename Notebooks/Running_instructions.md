# Running Instructions

## Project Overview

This project implements a Vehicle Type Classification System using Convolutional Neural Networks (CNNs) and Transfer Learning with MobileNetV2 on the Stanford Cars Dataset.

## Requirements

* Kaggle account
* Verified Kaggle account (required for GPU access)
* Internet connection enabled in Kaggle
* Stanford Cars Dataset by Classes Folder dataset

## Dataset

Dataset used:

Stanford Cars Dataset by Classes Folder

Source:
https://www.kaggle.com/datasets/jutrera/stanford-car-dataset-by-classes-folder

## Kaggle Configuration

1. Create a new Kaggle Notebook.
2. Upload or import the notebook:
   `vehicle_classification_cnn.ipynb`
3. Add the Stanford Cars dataset.
4. Open Settings.
5. Enable:

   * Accelerator: GPU T4 x2
   * Internet: ON

## Running the Notebook

Execute all notebook cells sequentially from top to bottom.

The notebook performs the following tasks:

1. Dataset loading
2. Dataset exploration
3. Image preprocessing
4. Data augmentation
5. Custom CNN training
6. MobileNetV2 transfer learning training
7. Model evaluation
8. Classification report generation
9. Confusion matrix generation
10. Error analysis
11. Saving output figures

## Generated Outputs

The notebook generates:

* Class distribution visualization
* Sample vehicle images
* Custom CNN accuracy/loss curves
* MobileNetV2 accuracy/loss curves
* Confusion matrix
* Classification report
* Model comparison table
* Misclassified image examples

Outputs are stored in:

`/kaggle/working/outputs`

## Notes

The trained `.keras` model files are not included in this repository because of their large size. They can be regenerated by executing the notebook.
